# KPK App - Claude Code Context

## What This Is

**Blendverse** - A Django manufacturing management system for Kinpak, Inc., a chemical blending facility in Montgomery, AL. Replaces legacy Excel systems with integrated MRP, real-time inventory, production scheduling, and quality control.

## Architecture At-a-Glance

```
[Sage 100 ERP] + [SharePoint Excel] + [IOT Tanks]
                    ↓
           [data_sync ETL worker]
                    ↓
              [PostgreSQL]
                    ↓
         [Django + Daphne + Redis]
                    ↓
    [Nginx reverse proxy (blue/green)]
                    ↓
              [Browsers]
```

**Production server:** `192.168.178.169` (kpkapp.lan)

## Key Components

| Component | Location | Purpose |
|-----------|----------|---------|
| Django app | `app/` | Web UI, APIs, WebSockets |
| ETL workers | `host-services/workers/` | data_sync, excel_worker, stream_relay |
| ETL modules | `local_machine_scripts/python_db_scripts/app_db_mgmt/` | Sage sync, table builders |
| Control Panel | `control-panel/` | Go CLI + GUI for service management |
| Docker configs | `docker-compose-*.yml` | DEV and PROD environments |

## Operational Commands

The `kpk` command is a PowerShell alias that runs **locally** and connects to the production server remotely. Run commands via `powershell -Command "kpk <command>"` or directly in PowerShell.

Key commands:

```
kpk status                    # Containers + services + loop function health
kpk loop status               # ETL function status only
kpk container logs <name>     # View container logs
kpk service logs <name>       # View host service logs (data_sync, excel_worker, etc.)
kpk service start/stop <name> # Manage host services
```

Container short names: `blue`, `green`, `nginx`, `db`, `redis`, `excel`, `ws4kp`

Host services: `data_sync`, `excel_worker`, `stream_relay`, `looper_health`

## ETL Health Verification

The `data_sync` worker writes status to `core_loopstatus` table after each function runs:
- **Function name** - which ETL function
- **Result** - Success/Failure
- **Timestamp** - when it last ran

`kpk status` shows this as:
- `[+] UP` - All functions healthy, recent timestamps
- `[!] DEGRADED` - Some errors but timestamps recent
- `[-] DOWN` - Stale (no updates in 5+ minutes)

## Django Layer Pattern

```
Views (web.py, api.py) → Services (business logic) → Selectors (queries)
```

- `core/selectors/` - Data retrieval (querysets, dicts)
- `core/services/` - Business logic, side effects, WebSocket broadcasts

## Important Tables

**User-managed (need backup):** `core_lotnumrecord`, `core_deskoneschedule`, `core_blendcountrecord`, etc.

**Sage-synced (read-only):** `ci_item`, `im_itemwarehouse`, `bm_billheader`, `po_*`, `so_*`

**Calculated (regenerated by ETL):** `bill_of_materials`, `component_shortage`, `timetable_run_data`

## Conventions

- **Lot numbers:** `[MonthLetter][YY][4-digit seq]` e.g., `A2500001` = Jan 2025, #1
- **Item prefixes:** `BLEND-*`, `CHEM-*`, `DYE-*`, `FRAGRANCE-*`
- **URLs:** lowercase-with-dashes
- **Python:** snake_case functions, CamelCase classes
- **JS:** camelCase modules, page initialization in `js/pageModules/`

## Deploying Changes

```
kpk git pull                  # Pull latest from origin/main
kpk git collectstatic         # Required for JS/CSS/static file changes
```

- **Python changes** (.py files): Auto-reload via Daphne - live immediately after `git pull`
- **Static file changes** (.js, .css, templates): Require `kpk git collectstatic` to copy into Docker volume

## Quick Troubleshooting

| Issue | Check |
|-------|-------|
| Data stale | `kpk status` or `kpk loop status` |
| Container down | `kpk container list` then `kpk container start <name>` |
| Service logs | `kpk service logs data_sync -n 100` |
| WebSocket issues | Verify Redis running, check Daphne logs |
