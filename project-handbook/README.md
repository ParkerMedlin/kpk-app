# Project Handbook

Accumulated knowledge for the KPK/Blendverse project.

## Structure

```
project-handbook/
├── feature-specs/     # Planned features (requirements → design → tasks)
└── troubleshooting/   # Documented problems and solutions
```

## Feature Specs

Each feature gets a folder with three documents following the spec-driven workflow:

1. `requirements.md` - What needs to be built (user stories, acceptance criteria)
2. `design.md` - How it will be built (components, data model, integration)
3. `tasks.md` - Step-by-step implementation checklist

See the `plan-new-feature` Claude skill for templates and workflow.

## Troubleshooting

Documented solutions to problems we've encountered. Each file covers one issue:

- Problem description
- Root cause
- Solution/fix
- How to verify it's resolved

Goal: Don't solve the same problem twice.
